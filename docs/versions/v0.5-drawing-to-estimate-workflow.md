# PAAX AI v0.5: Drawing-to-Estimate AI Workflow Foundation

## Overview
Version 0.5 introduces the "Drawing-to-Estimate" workflow. This bridges the gap between unstructured drawings (PDFs/images) and the deterministic Core Engine. PAAX v0.5 provides the *foundation* for this workflow, emphasizing mandatory human verification.

## Architecture & Data Flow
1. **Drawing Intake**: Users provide drawing metadata (and optionally a mock file).
2. **Drawing Analysis (Document Intelligence)**: The new `document-intelligence` service (port 8083) processes the drawing. If a Gemini API key is missing, it falls back to a realistic demo extraction.
3. **Extraction Candidates**: The service returns structured data (rooms, doors, windows, dimensions, and quantity candidates).
4. **Human Verification**: **MANDATORY STEP**. Users review, approve, reject, or edit quantity candidates in the `Gambar Kerja AI` workspace.
5. **BOQ Draft Preview**: Generates a draft Bill of Quantities from verified data only.
6. **RAB Handoff**: Verified drawing data is packaged into a `DrawingToRabContext` and passed to the `RAB & BOQ` module.
7. **Schedule Readiness**: The schedule module becomes aware that drawing data exists.
8. **Engineering Chat**: Chat assistant is aware of the drawing context and verification status.

## Key Principles & Limitations
- **No Automatic Final RAB**: The AI does *not* generate final RAB numbers directly. Final math remains the responsibility of the Core Engine.
- **Mandatory Verification**: AI extraction results are labeled as "candidates". They must be human-verified before being used in calculations.
- **Frontend Safe**: No API keys (like `GEMINI_API_KEY`) are exposed in the frontend. All AI interactions go through the backend services.
- **Fallback Mode**: If the real AI provider is unavailable or unconfigured, the system gracefully falls back to a deterministic demo extraction to allow workflow testing.

## New Services & Endpoints
**Document Intelligence Service** (`http://127.0.0.1:8083`):
- `GET /health`
- `POST /drawings/analyze`
- `POST /drawings/classify`
- `POST /drawings/extract`
- `POST /drawings/verify`
- `POST /drawings/boq-preview`

## LocalStorage State Keys
- `paax_drawing_files`: Stores uploaded/mocked drawing files.
- `paax_verified_quantities`: Stores human-verified quantity candidates.
- `paax_drawing_to_rab_context`: The handoff package sent from Gambar Kerja AI to the RAB module.

## Testing Steps
1. Navigate to **Gambar Kerja AI**.
2. Add a new drawing file (demo).
3. Click **Analyze Drawing**.
4. Verify candidates in the **Human Verification Workflow** section.
5. Click **Use Verified Drawing Data for RAB**.
6. Check the **RAB & BOQ** page for the "Verified Drawing Data Available" panel.

## Path to v0.6
Version 0.6 will implement the final drawing-based calculation in the Core Engine, merging the `DrawingToRabContext` directly into the deterministic RAB generation algorithm.
