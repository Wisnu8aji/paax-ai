# PAAX AI v0.2 (Demo) - Summary

## Overview
Version 0.2 was the initial proof-of-concept and interactive demo of the PAAX AI platform. It successfully proved the core hypothesis: AI can assist civil engineers in navigating drawings and generating standard calculations, provided the math is handled deterministically.

## Architecture
- **Frontend**: React + Vite (Single Page Application).
- **Backend**: Express.js (Node.js).
- **AI**: Direct integration with the Gemini API via Google AI Studio keys.
- **Data**: Mock AHSP (Analisa Harga Satuan Pekerjaan) data stored in static JSON files.

## Key Features Demonstrated
1. **RAB Tab**: Displayed a basic Bill of Quantities structure.
2. **Assistant Tab**: A chat interface where users could ask engineering questions and the bot would respond using a pre-defined system prompt.
3. **Drawing Tab**: Basic viewing of uploaded plans.
4. **Schedule Tab**: Simple Gantt chart representation.
5. **Rates Tab**: A lookup table for the mock AHSP data.

## Limitations Addressed in v0.3
- **Lack of Persistence**: v0.2 used local state or basic JSON files. It lacked a real database (Firestore) for multi-user, multi-project support.
- **Fragile AI Integration**: Direct API calls to Gemini lacked robust orchestration, tool calling, and fallback mechanisms.
- **Monolithic Frontend**: The Vite app became bloated.
- **Backend Constraints**: Node.js/Express is less suited for heavy deterministic numerical computations and data science tasks compared to Python.

## Legacy Code
The code for v0.2 has been archived but is kept for reference in `legacy/vite-v0.2/`. The core concepts, especially the strict separation of AI and deterministic math, heavily influenced the architecture of v0.3.
