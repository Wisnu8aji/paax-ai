# PAAX AI v0.4: Interactive Workspace & Core Engine Integration

## Overview
Version 0.4 focuses on making PAAX an interactive workspace rather than a static dashboard. It bridges the frontend UI (`apps/web`) with the deterministic calculation engine (`services/core-engine`) to simulate real production workflows using `localStorage` as a mock persistent state layer before a real database (Firebase/PostgreSQL) is introduced in later versions.

## Key Goals
1. **Interactive State Management**: 
   - Projects, RAB data, Schedules, and Files are now saved across page reloads using browser `localStorage`.
   - Users can create new projects, and the dashboard automatically reflects these updates.
2. **Core Engine Integration**: 
   - Standardized the API communication through `apps/web/src/lib/core-engine-client.ts`.
   - Connected the frontend directly to the `services/core-engine` FastAPI endpoints running locally on `http://127.0.0.1:8081`.
3. **Module Enhancements**:
   - **Proyek (Project Workspace)**: Full CRUD mock utilizing `localStorage`.
   - **RAB (Bill of Quantities)**: Connected to the core engine for calculation generation, recalculation based on modified margins, safety score analysis, and structural review algorithms.
   - **Schedule**: Multi-scenario generation (Normal, Fast, Optimal) and Delay Recovery strategies linked directly to backend algorithms.
   - **Laporan (Reporting)**: Core engine-powered Excel export functionality.
   - **Chat / AI Assistant**: Interactive mock interface demonstrating context-aware actions.
   - **Site Agent**: Mock field reporting and daily progress generation.
   - **Database AHSP**: Dynamic search and filtering for standard construction material and labor rates.
   - **Pengaturan (Settings)**: Real-time health check ping to the `core-engine`.

## Technical Implementation Notes
- **State Layer**: Avoided adding massive state management libraries (like Redux) for v0.4. Standard React `useState` coupled with a `LocalStorage` utility class provided enough persistence to make the prototype feel "real" to the user.
- **Data Flow**: Calculations must originate from the `core-engine`. The frontend React application acts solely as a thin presentation client. 
- **Error Handling**: Implemented robust error catching on all API calls, failing gracefully with alerts or fallback mock data when the backend is unreachable.

## Future Path to v0.5
- Replace `localStorage` with a real database (Firestore or PostgreSQL).
- Implement proper user authentication and multi-tenant logic.
- Integrate the Genkit AI Orchestrator to fully power the interactive chat features with real LLM responses.
- Implement the Document Intelligence pipeline to allow users to drag-and-drop a PDF drawing and have the system automatically seed the RAB.
